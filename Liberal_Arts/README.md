## 창의융합프로젝트(I)
1. 해시 함수는 단방향 함수이며, 해시 값을 이용해 원래 입력값을 복원할 수 없다.
    - 참
2. 블록체인 기술의 주요 특징으로 옳지 않은 것은?
    - 1. 암호학적 해시 함수를 활용하여 보안성을 강화한다.
    - 2. 데이터 수정이 용이하여 빠르게 변경할 수 있다. (X)
    - 3. 모든 거래 내역이 기록되는 공개 장부이다.
    - 4. 중앙 기관 없이 분산된 형태로 운영된다.
3. 해시 함수에 대한 설명으로 틀린 것은?
    - 1. 같은 입력값에 대해서는 항상 같은 해시 값이 나온다.
    - 2. 입력값이 다르면 해시 값도 달라야 한다.
    - 3. 임의의 길이를 가진 입력을 고정된 길이의 출력으로 변환한다.
    - 4. 해시 값에서 원래 입력값을 유추할 수 있어야 한다.
4. 암호학적으로 안전한 해시 함수의 활용 예로 옳지 않은 것은?
    - 1. 전자서명
    - 2. 데이터 압축 (X)
    - 3. 데이터 무결성 검증
    - 4. 패스워드 암호화
5. 80비트 보안 강도의 의미로 옳은 것은?
    - 1. 80개의 키를 사용하여 암호화가 이루어진다는 의미이다.
    - 2. 알고리즘의 취약성을 알아내려면 2의 80승 번의 연산이 필요하다는 의미이다. (O)
    - 3. 80개의 서로 다른 암호화 방식이 적용된다는 의미이다.
    - 4. 알고리즘이 80비트 크기의 데이터를 암호화한다는 의미이다.
6. 사용자 계정의 패스워드를 보호하는 기술로는 크게 암호화하는 방법과 해시값으로 만드는 방법이 사용된다.
    - 참
7. 사용자 패스워드를 해시값으로 저장할 때 salt를 추가하는 이유는 패스워드의 길이를 늘림으로써 보안을 강화하기 위해서이다.
    - 거짓
8. 암호학에서 salt는 패스워드 보안을 강화하기 위해 해시값 계산 시 추가되는 임의의 데이터이다.
    - 참
9. 사전 공격(Dictionary Attack)은 공격자가 미리 준비된 단어 목록을 사용하여 패스워드를 추측하는 방법이다.
    - 참
