## 창의융합프로젝트 (수요일 수업2)
### 0319 수업내용
1. DES 이후
    - 암호가 일반적으로 사용됨
    - 예전에는 군사/외교적 목적 -> 암호를 기반으로 메시지 보안, 인증, 서명 등 사용 -> 현금자급기, 패스워드, 전자상거래 등 암호 기반으로 안정성 보장
    - 금융 시스템을 중심으로 상업용 암호화의 이용, 컴퓨터 통신망을 이용한 문자 전송, 전자 자금 이체 등이 활성화
2. 커크호프 원칙
    - 1883년, 커크호프가 "키 이외에 암호 시스템의 모든 것이 공개되어도 안전해야 한다."
    - 표준화된 암호와 컴퓨팅 기기들을 사용하는 현대암호에서는 암호 알고리즘을 감추기가 매우 어려움
    - 현대암호에서는 암호 알고리즘을 공개하도록 하고 있음
    - 암호 방식의 안정성을 공개적으로 검토하게 함
3. AES 개요
    - 1997년 미국 NIST에서 DES를 대체할 대칭 암호 알고리즘 공개 모집
    - 2001년 블로암호인 라이달이 최종 AES로 채택
    - 암호화 과정
        - 키 : 128~256비트/ 평문입력: 128비트/ 암호문 출력 : 128비트
4. SPN 구조 (Substitution-Permutaition Network)
    - Subsitution(대체)과 permutaion(치환)의 반복 구조
        - S-Box를 이용하여 대체
        - P-Box를 이용하여 대체
    - 암호화 복수와 복호화 함수가 다르기 때문에 복호화되기 위해서는 라운드 함수가 역변환이 되어야 함
    - 하드웨어나 소프트웨어 구현 시 비용이 많이 듦
    - 대표적으로 AES
5. 블록 암호 알고리즘
    - DES
    - Triple DES
    - AES
    - IDEA (International Data Encryption Alogorithm)
    - 국내
        - SEED
        - LEA (Lightweight Encryption Algorithm)
6. 블록 암호 운용 모드 (알고 있어야할 듯 시험에 나올 수도)
    - 암호화 모드(Encryption Mode)
    - 긴 평문을 암호화하기 위해서 블록 암호 알고리즘을 반복해서 사용
    - 블록 암호 알고리즘을 반복하는 방법을 블록 암호의 운용 모드 라고 함
    - ECB / CBC / CFB / OFB / CTR 모드
7. ECB의 취약점
    - 동일한 평문 블록은 동일한 암호문 생성 -> 트래픽 분석 (traffic analysis) 용이
    - 암호문 블록의 삭제나 교체에 의한 평문 조작과 재전송 공격 가능
8. 정보보안산업기사/정보보안기사 블록암호의 운영모드

9. 대칭키 암호의 장점
    - 암호 알고리즘이 간단한 치환과 전치의 조합으로 구성됨
    - 암호화 연산 속도가 빨라서 효율적임
    - 데이터 처리량이 큼
    - 암호화 키의 크기가 공개키보다 상대적으로 적음
    - 알고리즘 내부 구조가 간단하여 시스템 환경에 적절한 암호 알고리즘 개발이 용이함
10. 대칭키 암호의 장점
    - 송신자와 수신자 간에 동일한 키를 공유해야 함
    - 많은 사람과 통신해야 하는 경우, 많은 키를 생성, 유지, 관리해야 함 (키 관리가 어려움)
11. 암호 알고리즘별 보안 강도
    - KISA 발표, 암호 알고리즘을 전수조사 공격을 통하여 깨는데 필요한 연산의 수

12. 공개키 암호란?
    - 공개키 암호는 송신자와 수신자가 다른 키를 사용하여 비밀 통신을 수행함
    - 즉, 암호화 키와 복호화 키가 다름
13. DH (Diffie-Hellman)
    - 1976년, 디피와 헬만이 처음으로 공개키 암호의 개념 발표
    - 기존 대칭키 암호 방식이 송신자 수신자 간 통신 전에 암/복호화 키를 서로 공유해야 하는 문제 지적
14. RSA (Rivest, Shamir, Adleman)

1. 정보시스템이란?
    - 정보시스템(Information System)
        - 데이터를 원하는 형태로 수정, 보완, 재정의하는 것을 목적으로 연관되거나 관련된 구성 요소들의 모임
        - 현대 사회와 기업의 운영에 있어 필수적인 도구
        - 정보시스템의 예시 : 인사관리시스템
        - 응용소프트웨어
            - 조직의 데이터를 처리하고 사용자를 지원하기 위해 설계된 소프트웨어
            - 목적 : 데이터를 정보로 변환
        - 응용소프트웨어 외
            - H/W 및 시스템 소프트웨어
            - 시스템 분석가에 의해 작성된 문서 및 교육자료
            - 시스템 전반에 걸친 업무에 대한 역할
            - 사용자, 운영자 및 관리자
        - 컴퓨터 기반 정보시스템과 관련된 요소들
            - 컴퓨터 기반 정보시스템 애플리케이션
                - 하드웨어
                - 시스템 소프트웨어
                - 문서와 교육 메뉴얼
                - 시스템 사용자
                - 통제
                - 특정 업무 역할
        - 정보시스템의 주요 기능
            - 데이터 수집
            - 데이터 저장
            - 데이터 처리
            - 정보 전달
            - 의사결정 지원
